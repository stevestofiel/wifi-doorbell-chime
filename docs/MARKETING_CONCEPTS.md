# Marketing Concepts

Working product framing, public-facing copy, and early commercial ideas for the
smart chime system.

## Product Positioning

A local-first smart chime system for doors, driveways, mailboxes, cameras,
sensors, and home automation events.

The core idea:

```text
Every event gets its own sound.
```

The product starts as a customizable Wi-Fi chime, then expands into a small
local alert system with remote sensors, automation-friendly endpoints, event
rules, and future long-range LoRa support.

## Short Description

A local-first smart chime system that plays custom sounds from doorbells,
sensors, cameras, and home automation events. Use it as a standalone Wi-Fi
chime, or expand it with remote sensors for doors, motion, mailboxes,
driveways, and outbuildings.

## Polished Description

A customizable smart chime for the whole property. Upload your own sounds, map
different sensors to different alerts, and trigger chimes from buttons, motion
sensors, home automation, or camera events. It works locally over your network,
with no cloud required for the core experience.

## Plainspoken Description

It is a Wi-Fi chime that does more than ring. It can play different sounds for
the front door, mailbox, driveway, shop, gate, motion sensor, or automation
trigger. Everything runs locally on your network, and you stay in control.

## Integration Messaging

The integration story should create interest without implying official
partnerships before integrations are tested and documented.

Safe public wording:

```text
Automation-friendly endpoints for platforms like Home Assistant, Homey,
UniFi Protect workflows, MQTT, and webhooks.
```

Alternate wording:

```text
Designed to fit into local smart-home ecosystems.
```

Visual concept:

```text
Smart Chime at center
  -> floating integration tiles:
     Home Assistant
     Homey
     UniFi Protect
     MQTT
     Webhooks
     Remote Sensors
     LoRa Range Pod
```

Use official third-party logos only when usage rights and integration claims are
clear. Until then, generic integration tiles are safer for public marketing.

## Product Family Language

Avoid presenting the system as too many separate products. Prefer one system
with deployment options.

```text
Core:
  Wi-Fi Chime

Expansion:
  Remote Sensor
  Range Pod
  LoRa Gateway Chime
```

Use cases:

```text
Front door
Mailbox
Driveway
Gate
Shop
Shed
Outbuilding
Camera event
Home automation trigger
```

## Notes For Later

- Validate actual Home Assistant, Homey, UniFi Protect, MQTT, and webhook
  workflows before making firm compatibility claims.
- Create product photos or renders after enclosure direction stabilizes.
- Keep "local-first" prominent, but avoid overpromising security until auth and
  transport decisions are hardened.
- Revisit naming once the physical product shape is clearer.
